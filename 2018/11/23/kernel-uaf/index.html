Windows Kernel Driver Exploitation (UAF)

Prerequisite

  • HackSys Driver
  • virtualKD
  • WinDBG

UAF

  • What is UAF?
    • Use After Free specifically refers to the attempt to access memory after it has been freed, which can cause a program to crash or, in the case of a Use-After-Free flaw, can potentially result in the execution of arbitrary code or even enable full remote code execution capabilities

Vulnerable code

  • free the pointer without assign to null

steps to exploit

  • spray memory pool
  • create holes in pool

used functions in Drivers

  • AllocateUaFObject
  • FreeUaFObject
  • UseUaFObject
  • AllocateFakeObject

Interact with the functions

  • type definition
  • CreateFile interact with the kernel driver
  • DeviceIoControl will interact with the function in Kernel Driver , 0x222013 is the address of AllocateUaFObject
  • to find address of AllocateUaFObject , you can calculate with CTRL_CODE which is defined in headers.
  • calculate with following command in python shell.

hex((0x00000022 << 16) | (0x00000000 << 14) | (0x802 << 2) | 0x00000003)

  • Run above powershell script in debugger machine.
  • lab set up can be found in following
    • https://rootkits.xyz/blog/2017/06/kernel-setting-up/
    • http://www.fuzzysecurity.com/tutorials/expDev/14.html
  • check the Chunks

Spray Kernel Memory Pool

  • IoCo-PoolSpray will spray the 5000 chunks

Create Holes in spray block

  • the script is creating 2500 holes
  • callback funtion will use one of the holes that we created .
  • so we need to put shellcode in this holes using FakeUafObject function.

Control EIP

  • above code return EIP value with AAAA
